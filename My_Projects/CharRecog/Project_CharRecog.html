Single Digit Number Recognition Project

This is a project I decided to try after seeing the algorithm online. It is a simple algorithm which produces good results.

It is also computationally inexpensive and thus very fast in comparison to other bulkier algorithms.

The Algorithm

Like all other data driven methods, this one consists of two phases, a training phase and a testing phase.

Training

  1. Greyscaling
  2. Bitmapping
  3. Storing

Greyscaling: The training image set is first greyscaled since our output does not depend on color. This makes the testing phase easier and the algorithm perform better.

Bitmapping: This is the step where the image is converted to a matrix of numbers. The color of each pixel in the image corresponds to a floating point number. Converting each pixel to a number will produce a matrix of numbers that can be used to represent that image, that matrix is called a bitmap.

Storing: Finally, once all the images are converted to their respective bitmaps they are stored and used as the trained model for prediction in the testing phase.

Testing

  1. Greyscaling/Bitmapping
  2. minDist Similarity Measure
  3. k-NN comparison

Greyscaling/Bitmapping: Once a image is selected for testing it must go through steps 1 and 2 in the training phase before it can be used for prediction

minDist Similarity Measure: The test bitmap is then compared against each training bitmap using the minDist method. The minDist method is simply the Euclidean Distance between the two bitmaps(matrices).

k-NN comparison: This step uses k-Nearest Neighbours algorithm to select a predicted value from the similarity measure values produced from the previous step.
